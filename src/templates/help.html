1. What is MacWatch?

MacWatch is a system health dashboard for your Mac. It monitors CPU, memory, disk, network connections, and security indicators — all from a single interface. It runs entirely on your Mac at localhost:8077.

2. Tabs

MacWatch has five tabs:

TabPurpose
DashboardOverview of system health: CPU/memory/disk gauges, top 5 CPU consumers, top 5 network talkers, and alert summary. A quick snapshot of everything.
ProcessesFull sortable table of all active processes with CPU and memory usage. Click column headers to sort. System-wide CPU, memory, and disk stats shown at top. Click any row to see process details.
NetworkAll applications with active network connections, grouped by threat score. Expand cards to see individual connections with DNS, WHOIS, and threat flags. Includes search and filters.
AlertsSystem alerts grouped by category (Network, CPU, Memory, Disk) with severity indicators and educational explanations for each alert type.
AI AnalysisOptional AI-powered assessment of security, performance, and system health. Supports Ollama (local), Claude API, Claude Web, and other providers.

Process Names

For interpreted languages (Python, Java, Node.js, Ruby, etc.), MacWatch extracts the actual script or module name from the command line instead of showing the generic interpreter name. For example:

  • python3 /path/to/run.py shows as run.py instead of "python3"
  • java -jar myapp.jar shows as myapp.jar instead of "java"
  • node server.js shows as server.js instead of "node"
  • python3 -m flask run shows as flask instead of "python3"

The original binary name is shown as a muted hint next to the display name.

8. FAQ

Q: Does MacWatch send my data anywhere?
No. It runs on localhost:8077 only. No data leaves your machine (unless you use a cloud AI provider on the AI Analysis tab).

Q: Why do some IPs show "(no rDNS)"?
Not all IPs have reverse DNS records. Check the whois org name instead.

Q: How accurate is threat scoring?
Scores are heuristic indicators, not security verdicts. A red score means "investigate," not "malware."

Q: Why does Activity Monitor show more processes?
The Processes tab shows all processes with CPU usage above 0%. Activity Monitor includes idle processes too. The Network tab only shows processes with active network connections.

Q: Why does my Python/Java app show a generic name?
MacWatch extracts script and module names from command-line arguments for common interpreters. If the name still looks generic, the process may have been launched with an unusual command pattern.

Q: Can MacWatch block connections?
No. Use macOS Firewall or Little Snitch for blocking.

Q: Does AI Analysis send my data externally?
It depends on the provider. Ollama runs entirely on your Mac — no data leaves your machine. Cloud providers (Claude API, Claude Web, etc.) send your MacWatch data to their API for analysis. AI analysis is completely optional and only happens when you explicitly click the button.

Q: How do I set up AI Analysis?
Ollama (local, free, private): Install Ollama (brew install ollama), start it (ollama serve), pull a model (ollama pull llama3.1), then select Ollama on the AI Analysis page. No API key needed — all data stays on your machine.
Claude API (requires prepaid API credits): Set export ANTHROPIC_API_KEY="your-key" before starting MacWatch. Requires credits at console.anthropic.com.
Claude Web (uses your Pro/Max subscription): Set CLAUDE_SESSION_KEY and CLAUDE_ORG_ID in a .env file at the project root. Get these from claude.ai → browser DevTools → Application → Cookies (sessionKey) and the URL path (/api/organizations/{orgId}/...).
The AI Analysis page will show setup instructions if a provider is not configured.
