1. What is MacWatch?

MacWatch shows you what your Mac applications are doing on the network. It answers:

  • Which apps have open network connections?
  • What servers are they connecting to?
  • Is any of it suspicious?

It runs entirely on your Mac. No data is sent anywhere.

7. FAQ

Q: Does MacWatch send my data anywhere?
No. It runs on localhost:8077 only. No data leaves your machine.

Q: Why do some IPs show "(no rDNS)"?
Not all IPs have reverse DNS records. Check the whois org name instead.

Q: How accurate is threat scoring?
Scores are heuristic indicators, not security verdicts. A red score means "investigate," not "malware."

Q: Can MacWatch block connections?
No. Use macOS Firewall or Little Snitch for blocking.

Q: What is the Analysis page?
The Analysis page shows all current alerts grouped by severity with detailed educational explanations for each alert type. It also includes an optional AI-powered analysis feature.

Q: Does AI Analysis send my data externally?
It depends on the provider. Ollama runs entirely on your Mac â€” no data leaves your machine. Cloud providers (Claude, etc.) send your MacWatch data to their API for analysis. AI analysis is completely optional and only happens when you explicitly click the button.

Q: How do I set up AI Analysis?
Ollama (local, free, private): Install Ollama (brew install ollama), start it (ollama serve), pull a model (ollama pull llama3.1), then select Ollama on the Analysis page. No API key needed â€” all data stays on your machine.
Claude (cloud): Set export ANTHROPIC_API_KEY="your-key" before starting MacWatch.
The Analysis page will show setup instructions if a provider is not configured.
